# Security Policy — NEXUS Protocol

## Supported Versions

| Version | Support |
|---|---|
| 4.1.x | ✅ Active |
| 4.0.x | ✅ Security fixes |
| < 4.0 | ❌ No support |

---

## Reporting a Vulnerability

**Do NOT open a public issue for security vulnerabilities.**

Doing so could expose the problem before a fix is available.

**How to report:**
1. Go to the **Security** tab in the GitHub repository
2. Click **"Report a vulnerability"**
3. Or contact the maintainer directly through GitHub

**What to include in your report:**
- Affected nxlang version
- Description of the vulnerability
- Steps to reproduce it
- Potential impact
- If you have a proposed fix, include it

**What to expect:**
- Confirmation of receipt within **48 hours**
- Impact assessment within **7 days**
- Fix published within **7 days** for critical issues
- Public credit in the release (if you wish)

## nxlang Security Model

`nxlang` is a parsing and validation library. Its security model:

| Protection | Description |
|---|---|
| Size limit | Files > 500KB are rejected before processing |
| Line limit | Maximum 2000 lines per file |
| Null byte detection | Blocks binary injection attempts |
| String-aware tokenizer | Tokens inside `"quotes"` are never evaluated |
| No code execution | nxlang never executes the code it validates |
| No network calls | Purely local processing |
| No filesystem access in `core/` | The core module does not read or write files |

---

## Scope

This is the security scope for `nxlang`:

**In scope:**
- Vulnerabilities in the parser or validator that allow validation bypass
- CLI vulnerabilities that affect the user's filesystem
- Dependencies with known CVEs

**Out of scope:**
- Vulnerabilities in code generated by AI using NEXUS (that's the model's responsibility)
- Denial of service attacks via very large files (already mitigated with the 500KB limit)
